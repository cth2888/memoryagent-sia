**Traditional Modeling:**

Conventional autoregressive language models represent a token sequence of length N as:

$$p(\mathbf{x}_{1:N}) = p(x_1) \prod_{n=2}^{N} p(x_n \mid \mathbf{x}_{1:n-1})$$

This requires caching all historical tokens, leading to quadratic complexity in computation and memory usage.

**Memory Agent Architecture:**

Memory Agent reformulates the decoding process by introducing a memory-augmented, segment-wise strategy:

  • **Chunking:** The input is divided into K sequential chunks $$(c^1, c^2, ..., c^K)$$, each of length ‚â§ C.
  • **Iterative Processing:** Chunks are processed sequentially, keeping computation per chunk constant.
  • **Memory Augmentation:** A latent memory state $$m^k$$ is updated after each chunk.
$$p(\mathbf{x}_{1:N}) = \sum_{\mathbf{m}^{1:K-1}} \prod_{k=1}^{K} \underbrace{p(\mathbf{c}^k \mid \mathbf{m}^{k-1})}_{\text{read}} \cdot \underbrace{p(\mathbf{m}^k \mid \mathbf{c}^k, \mathbf{m}^{k-1})}_{\text{write}}$$
